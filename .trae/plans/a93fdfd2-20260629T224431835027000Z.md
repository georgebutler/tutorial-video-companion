# Plan: Rename and Open-Source `tutorial-video-companion`

## Goal
Rename the app from `webspatial-shop-script` to `tutorial-video-companion`, prepare it for open-source use with real project docs/license metadata, and move the current WebSpatial tutorial copy into a gitignored private content area while keeping a tracked public sample so fresh clones still build.

## Key Decisions
- Keep generic tutorial data types tracked in `src/data/tutorial.ts`.
- Move the current project-specific copy from `src/data/videoCopy.ts` into `src/content/private/tutorialContent.ts`.
- Add `src/content/private/` to `.gitignore` so the current tutorial content is local/private.
- Add a tracked generic sample at `src/content/public/tutorialContent.ts` so the app works out of the box for open-source users.
- Update the app to import tutorial data through a neutral alias, `@tutorial-content`, which resolves to private content when present and public sample content otherwise.
- Use an MIT license with a generic contributor copyright line unless the project owner later wants a specific name/org.
- Avoid adding fake repository/homepage URLs to `package.json`; document that they should be added once the GitHub repo URL is known.

## Files to Modify
- `package.json`
  - Rename package to `tutorial-video-companion`.
  - Remove `private: true`.
  - Add OSS-friendly metadata: `description`, `license`, and `keywords`.
- `package-lock.json`
  - Update root package name after package metadata changes.
- `index.html`
  - Change the document title to `Tutorial Video Companion`.
  - Add a basic description meta tag.
- `vite.config.ts`
  - Change `base` to `/tutorial-video-companion/`.
  - Add a conditional alias for `@tutorial-content`:
    - use `src/content/private/tutorialContent.ts` if it exists,
    - otherwise use `src/content/public/tutorialContent.ts`.
- `tsconfig.app.json`
  - Add a matching TypeScript `paths` entry for `@tutorial-content`, with private first and public fallback second.
- `src/App.tsx`
  - Change the data import from `@/data/videoCopy` to `@tutorial-content`.
  - Rename the storage key to `tutorial-video-companion-progress`.
  - Add a legacy read fallback for `webspatial-shop-recording-companion` so local progress is not immediately lost.
- `.gitignore`
  - Add `src/content/private/` and optionally `src/content/**/*.local.ts`.
- `README.md`
  - Replace the default Vite README with an open-source README explaining purpose, setup, scripts, content customization, private content workflow, deployment base path, and license.
- `src/data/videoCopy.ts`
  - Remove this tracked project-specific content file after moving its contents to the ignored private content file.

## Files to Create

- `src/content/private/tutorialContent.ts`
  - Ignored local/private content file.
  - Will receive the current contents of `src/data/videoCopy.ts`.
- `LICENSE`
  - MIT license text.

1. Add `src/content/public/tutorialContent.ts` with a short generic tutorial sample that exercises the app UI: spoken narration, show/action lists, prompt/command block, verify block, recording notes, and editing notes.
2. Create `src/content/private/tutorialContent.ts` by moving the current `src/data/videoCopy.ts` content into it unchanged except for any import-path adjustments needed.

