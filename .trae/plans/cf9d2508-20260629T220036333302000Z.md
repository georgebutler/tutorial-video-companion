# Plan: Move Video Copy Into One Review File

## Goal
Make the video script/copy easy to review by moving all reviewable video copy into one dedicated file.

Currently, the copy already lives mostly in `src/data/tutorial.ts`, but that file also contains structural TypeScript types. The cleaner review experience is:

- `src/data/videoCopy.ts` = all video/script copy
- `src/data/tutorial.ts` = data-shape TypeScript types only

UI chrome labels such as `Copy`, `Recorded`, `Recording Companion`, `Reset progress`, `SAY`, and `SHOW` should remain in components for now because they are app interface labels, not video script copy.

## Files to Modify

- `src/data/videoCopy.ts` (new file)
- `src/data/tutorial.ts`
- `src/App.tsx`

`src/components/TutorialContent.tsx` should keep importing types from `src/data/tutorial.ts` and should not need behavior changes.

## Implementation Steps

1. **Create the single review file**
   - Add `src/data/videoCopy.ts`.
   - Add a short top-of-file comment such as:
     ```ts
     // All reviewable video script copy lives in this file.
     ```
   - Import the data type:
     ```ts
     import type { TutorialSegment } from "@/data/tutorial";
     ```

2. **Move all video/script copy into `videoCopy.ts`**
   Move these exports from `src/data/tutorial.ts` to `src/data/videoCopy.ts` unchanged:
   - `shopProjectPath`
   - `tutorialCommands`
   - `tutorialOverview`
   - `tutorialSegments`

   This includes all segment titles, time ranges, goals, narration paragraphs, prompt text, action lists, verification lists, recording notes, and editing notes.

3. **Leave only structural types in `tutorial.ts`**
   Keep only:
   - `TutorialCue`
   - `TutorialAudience`
   - `TutorialBlock`
   - `TutorialSegment`

4. **Update app data import**
   - In `src/App.tsx`, change:
     ```ts
     import { tutorialSegments } from "@/data/tutorial";
     ```
     to:
     ```ts
     import { tutorialSegments } from "@/data/videoCopy";
     ```

5. **Keep component type imports unchanged**
   - `src/components/TutorialContent.tsx` can continue importing the structural types from `@/data/tutorial`.
