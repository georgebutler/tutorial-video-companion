# Plan: Make Prompts Consistent, Code-Like, and Copyable

## Request summary

The user wants the script text and presentation to be more consistent, and specifically notes that prompts cannot be copied/pasted easily. Prompt and command content should be presented as proper code blocks with reliable copy behavior.

This is a refinement of the existing recording companion UI, not a new feature area.

## Current findings

- `src/data/tutorial.ts` already models prompts and commands correctly as `{ type: "prompt" | "run"; title: string; code: string }`.
- `src/components/TutorialContent.tsx:103` has the central `CodeBlock({ code })` component.
- `src/components/TutorialContent.tsx:133` routes all prompt/run blocks through `CodeBlock`.
- `src/components/TutorialContent.tsx:103-108` currently renders a semantic `<pre><code>` block, but it has no copy button and uses wrapping classes that make long prompts hard to select.
- `src/index.css:40-43` applies `overflow-wrap: anywhere` globally to both `pre` and `code`, which can visually fragment prompts and make them feel less like stable code blocks.

## Recommended implementation

### 1. Upgrade `CodeBlock` in `src/components/TutorialContent.tsx`

Convert the existing `CodeBlock` into a copyable code block component that includes:

- A bordered container with a header row.
- A small label: `Prompt` for prompt blocks and `Command` for run blocks.
- A visible `Copy` button.
- Transient success state: button text changes to `Copied` after a successful copy.
- `navigator.clipboard.writeText(code)` to copy the exact source string.
- Accessible button labeling such as `aria-label="Copy prompt"` or `aria-label="Copy command"`.

Implementation notes:

- Import `useState` from React in `src/components/TutorialContent.tsx`.
- Keep this component in the same file; do not create a new file.
- Use the existing `Button` component.
- Do not make the copy control icon-only; visible text is clearer and more accessible.

Target shape:

```tsx
function CodeBlock({ code, label }: { code: string; label: "Prompt" | "Command" }) {
  const [copied, setCopied] = useState(false);

  async function handleCopy() {
    await navigator.clipboard.writeText(code);
    setCopied(true);
    window.setTimeout(() => setCopied(false), 1600);
  }

  return (
    <div className="overflow-hidden rounded-lg border bg-muted/60">
      <div className="flex items-center justify-between gap-3 border-b bg-background/80 px-3 py-2">
        <span>{label}</span>
        <Button type="button" onClick={handleCopy} aria-label={`Copy ${label.toLowerCase()}`}>
          {copied ? "Copied" : "Copy"}
        </Button>
      </div>
      <pre>
        <code>{code}</code>
      </pre>
    </div>
  );
}
```

The final code should include error-safe copy handling so failed clipboard writes do not break the UI.

### 2. Pass content type from `TutorialBlockCard`

Update `src/components/TutorialContent.tsx:133` from:

```tsx
<CodeBlock code={block.code} />
```

to:

```tsx
<CodeBlock code={block.code} label={block.type === "prompt" ? "Prompt" : "Command"} />
```

This keeps the outer cue labels consistent:

- `PASTE` = action the presenter should take.
- `Prompt` = what the code block contains.
- `RUN` = action the presenter should take.
- `Command` = what the code block contains.

### 3. Fix wrapping so prompts behave like code blocks

Update `src/index.css` so global wrapping no longer affects block code.

Current rule:

```css
pre,
code {
  overflow-wrap: anywhere;
}
```

Recommended rule:

```css
:not(pre) > code {
  overflow-wrap: anywhere;
}
```

Then let `CodeBlock` control block formatting with Tailwind classes:

- `pre`: `max-h-56 overflow-auto p-3 text-xs leading-6 text-foreground sm:text-sm`
- `code`: `block whitespace-pre font-mono`

This makes long prompts horizontally scrollable instead of fragmented, while inline code remains resilient.

### 4. Presentation consistency

Keep the existing cue labels and colors:

- `SAY`
- `NOTE`
- `PASTE`
- `RUN`
- `VERIFY`

Do not rename `PASTE` back to `PROMPT`; `PASTE` is the presenter action. The inner code block label should provide the content type.

Ensure prompt and command blocks share the same structure:

1. cue section badge/title
2. code block header with label and copy button
3. scrollable code area
