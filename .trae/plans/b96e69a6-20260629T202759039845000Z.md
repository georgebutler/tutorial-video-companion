# Plan: Remove Rounded Corners and Standardize Page Styling

## Goal
Make the whole page visually consistent with **square corners everywhere**. Remove remaining `rounded-*` styling from local page components and shared UI primitives so cards, buttons, badges, code blocks, progress bars, and panels all use the same squared-off visual language.

This is a styling consistency pass only. No data-model changes are needed.

## Current Rounded-Corner Sources
Remaining rounded styles were found in:

- `src/components/TutorialContent.tsx`
  - `CodeBlock` wrapper: `rounded-md`
  - `TutorialSegmentCard`: `rounded-md`
  - progress bar track/fill: `rounded-full`
- `src/components/ui/card.tsx`
  - base `Card`: `rounded-xl`
  - image rounding: `rounded-t-xl`, `rounded-b-xl`
  - `CardHeader`: `rounded-t-xl`
  - `CardFooter`: `rounded-b-xl`
- `src/components/ui/badge.tsx`
  - badge base: `rounded-4xl`
- `src/components/ui/button.tsx`
  - base and size variants include `rounded-md` and arbitrary radius classes
- `src/index.css`
  - global radius tokens still resolve to non-zero values

## Implementation Steps

### 1. Zero global radius tokens

Edit `src/index.css`:

- Set all theme radius tokens to `0`:
  - `--radius-sm`
  - `--radius-md`
  - `--radius-lg`
  - `--radius-xl`
  - `--radius-2xl`
  - `--radius-3xl`
  - `--radius-4xl`
- Set root `--radius: 0`.

This provides a global safeguard if any token-based radius usage remains.

### 2. Make shared cards square and consistent

Edit `src/components/ui/card.tsx`:

- Remove all card/header/footer `rounded-*` classes.
- Remove child image rounding classes from the base card.
- Replace the current ring/shadow default with a consistent square panel style:
  - `border border-border bg-card shadow-none`
- Keep existing spacing and layout behavior.

This makes `RecordingCompanion`, `TutorialSegmentCard`, `ChecklistCard`, `InfoListCard`, and `Production reference` visually consistent by default.

### 3. Make buttons square globally

Edit `src/components/ui/button.tsx`:

- Remove base `rounded-md`.
- Remove all size-specific rounded classes:
  - `in-data-[slot=button-group]:rounded-md`
  - `rounded-[min(var(--radius-md),8px)]`
  - `rounded-[min(var(--radius-md),10px)]`
- Keep padding, height, focus rings, variants, and hover states unchanged.

This applies to copy buttons, sidebar nav buttons, and reset button.

### 4. Make badges square globally

Edit `src/components/ui/badge.tsx`:

- Remove `rounded-4xl` from the badge base class.
- Keep border, color, icon spacing, and focus styles unchanged.

This makes time, priority, and cue labels rectangular instead of pill-shaped.

### 5. Remove local rounded classes from tutorial content

Edit `src/components/TutorialContent.tsx`:

- Remove `rounded-md` from `CodeBlock`.
- Remove `rounded-md` from `TutorialSegmentCard` so the shared `Card` controls shape.
- Remove `rounded-full` from the progress bar track and fill.
- Keep active segment styling, key/optional styling, borders, and layout intact.

The progress bar should become square-ended:

```tsx
className="h-2 overflow-hidden bg-secondary"
className="h-full bg-primary transition-all"
```

### 6. Keep a consistent border model

Use the shared `Card` style as the single default card boundary:

```tsx
border border-border bg-card shadow-none
```

Keep existing segment state classes because they now work consistently against the base border:

- Active: `border-primary bg-primary/[0.03] ring-2 ring-primary/20`
- Key: `border-primary`
- Optional: `border-dashed`

Do not reintroduce local rounded overrides.
